plain

PROMPT 9A - "Kitchen Display System"

Berdasarkan SRS v4.0 UC-KIT-01, UC-KIT-02, UC-KIT-03, buatkan:

1. CONTROLLER: application/controllers/Kitchen.php
   - index() → halaman KDS
   - orders() → AJAX polling endpoint (POST /api/kitchen/orders)
   - accept() → terima pesanan (update status)
   - update_status() → ubah status item (diterima→dimasak→siap)
   - cancel_item() → tandai item tidak tersedia

2. SMART POLLING (assets/js/kitchen.js):
   - Interval: 5000ms default
   - Exponential backoff: 5s → 10s → 20s (max) saat error
   - Deduplication: flag is_requesting, skip jika request pending
   - Request: {last_id, last_timestamp}
   - Response: {updated, data[], last_id, last_timestamp}
   - Delta: server hanya return id > last_id
   - Rate limit: max 1 req/3 detik per session

3. UI SPECIFICATIONS:
   - Grid: 3 kolom (desktop ≥1280px), 2 kolom (tablet), 1 kolom (mobile)
   - Sticky header: jumlah pesanan aktif, rata-rata waktu tunggu, tombol batch accept, mute
   - Card pesanan: fixed height 280px
     * Header: kode meja (bold 18px), nomor pesanan (monospace), waktu masuk, timer durasi (HH:MM:SS, merah jika >15 menit)
     * Body: list item (nama, qty, catatan max 1 line), scrollable jika >4
     * Footer: tombol status bertingkat (Terima → Sedang Dimasak → Siap), min 44px height
   - Highlight pesanan baru (<2 menit): border 3px #FFC107, background kuning muda, pulse animation
   - Sound alert: beep 500ms (800Hz), HARUS di-enable via user interaction pertama, mute status di LocalStorage

4. STATUS FLOW:
   - Diterima → Dimasak → Siap (tidak boleh skip)
   - Undo: 30 detik setelah update (1x per item)
   - Batch accept: checkbox multiple card → "Terima Semua"
   - Auto-refresh indicator: dot hijau/merah pojok kanan atas

5. CANCEL ITEM:
   - Dropdown "Tidak Tersedia" per item
   - Modal alasan (opsional, max 100 char)
   - Validasi: hanya jika status masih 'diterima'
   - Update order subtotal (kurangi harga item batal)
   - Notifikasi ke pelanggan via polling

BUSINESS RULES:
- BR-17: FIFO ordering (created_at)
- BR-18: Hanya lihat item diterima/dimasak
- BR-19: Tidak boleh double accept
- BR-20: Status berurutan, tidak boleh skip
- BR-22: Semua item aktif siap → order status siap
- BR-23: Batal hanya jika status diterima
